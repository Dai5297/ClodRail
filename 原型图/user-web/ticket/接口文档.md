## 7. 座位选择

### 接口信息
- **URL**: `/api/seats/available`
- **方法**: `GET`
- **描述**: 获取指定车票和座位类型的可用座位信息

### 请求参数
通过URL查询参数传递：
- `ticketId`: 车票ID (必填) - 从车票搜索结果中获取的车票唯一标识符，用于确定具体的车次和路线
- `seatType`: 座位类型 (必填) - 座位类型代码，0-商务座，1-一等座，2-二等座

### 请求示例
```
GET /api/seats/available?ticketId=1&seatType=2
```

### 响应格式
```json
{
    "code": 200,
    "message": "success",
    "data": {
        "carriageNumber": 2,
        "seatType": "二等座",
        "availablePositions": ["A", "B", "C", "D", "F"],
        "seats": [
            {
                "seatNumber": "01A",
                "position": "A",
                "status": "available"
            },
            {
                "seatNumber": "01B",
                "position": "B",
                "status": "occupied"
            },
            {
                "seatNumber": "01C",
                "position": "C",
                "status": "available"
            }
        ]
    }
}
```

### 响应参数说明
- `code`: 响应状态码 - 200表示成功，其他值表示错误
- `message`: 响应消息 - 描述请求处理结果
- `data`: 响应数据对象
  - `carriageNumber`: 车厢号 - 当前查询的车厢编号
  - `seatType`: 座位类型名称 - 座位类型的中文描述（如"商务座"、"一等座"、"二等座"）
  - `availablePositions`: 可用座位位置列表 - 该座位类型下所有可能的座位位置代码数组
    - 一等座通常包含：["A", "B", "E", "F"]
    - 二等座通常包含：["A", "B", "C", "D", "F"]
    - 商务座通常包含：["A", "C", "F"]
  - `seats`: 具体座位信息列表 - 包含每个座位的详细状态
    - `seatNumber`: 座位号 - 完整的座位编号，格式为"行号+位置"（如"01A"表示第1排A座）
    - `position`: 座位位置 - 座位的位置代码（A、B、C、D、E、F等）
    - `status`: 座位状态 - 座位的当前状态
      - "available": 可预订
      - "occupied": 已被占用
      - "locked": 被锁定（正在预订中）
      - "maintenance": 维护中不可用

### 2. 车票搜索接口

**接口描述**: 根据出发地、目的地和日期搜索可用车票

**请求信息**:
- **URL**: `POST /api/tickets/search`
- **请求方式**: POST
- **Content-Type**: application/json

**请求参数**:
```json
{
  "originStationId": 1,
  "destinationStationId": 4,
  "departureDate": "2024-01-15",
  "returnDate": "2024-01-20"
}
```

**参数说明**:
- `originStationId`: 出发站ID (必填)
- `destinationStationId`: 到达站ID (必填)
- `departureDate`: 出发日期，格式：YYYY-MM-DD (必填)
- `returnDate`: 返程日期，格式：YYYY-MM-DD (可选)

**响应信息**:
```json

**字段说明**:

    - `destinationStation`: 到达站信息
  - `timeInfo`: 时间信息
    - `startTime`: 出发时间

    - `seatTypeName`: 座位类型名称
    - `price`: 价格
    - `totalSeats`: 总座位数
    - `remainingSeats`: 剩余座位数
    - `inventoryStatus`: 库存状态 (0-充足, 1-紧张, 2-售空)
    - `inventoryStatusName`: 库存状态名称

## 错误码定义

| 错误码 | 错误信息 | 说明 |
|--------|----------|------|
| 200 | success | 请求成功 |
| 400 | 参数错误 | 请求参数不正确 |
| 404 | 未找到数据 | 没有找到符合条件的车票 |
| 500 | 服务器内部错误 | 系统异常 |

## 前端调用示例

### JavaScript调用示例

```javascript
// 1. 获取车站列表
async function loadStations() {
    try {
        const response = await fetch('/api/stations');
        const result = await response.json();
        
        if (result.code === 200) {
            populateStationSelect('origin', result.data);
            populateStationSelect('destination', result.data);
        }
    } catch (error) {
        console.error('加载车站数据失败:', error);
    }
}

// 2. 搜索车票
async function searchTickets() {
    const searchParams = {
        originStationId: document.getElementById('origin').value,
        destinationStationId: document.getElementById('destination').value,
        departureDate: document.getElementById('departure-date').value,
        returnDate: document.getElementById('return-date').value,
        pageNum: 1,
        pageSize: 10
    };
    
    try {
        const response = await fetch('/api/tickets/search', {
            method: 'POST',
            headers: {
                'Content-Type': 'application/json'
            },
            body: JSON.stringify(searchParams)
        });
        
        const result = await response.json();
        
        if (result.code === 200) {
            displaySearchResults(result.data);
        } else {
            alert(result.message);
        }
    } catch (error) {
        console.error('搜索车票失败:', error);
        alert('搜索失败，请稍后重试');
    }
}
```

## 注意事项

1. **数据验证**: 前端需要验证出发地和目的地不能相同
2. **日期格式**: 日期格式统一使用 YYYY-MM-DD
3. **分页处理**: 搜索结果支持分页，默认每页10条记录
4. **错误处理**: 需要对API调用异常进行适当的错误处理
5. **缓存策略**: 车站列表数据可以考虑缓存，减少重复请求
6. **响应时间**: 车票搜索可能耗时较长，建议添加加载提示
