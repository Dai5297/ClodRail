# 订单相关 API 接口文档

## 1. 创建订单

### 接口信息

- **URL**: `/api/orders/create`
- **方法**: `POST`
- **描述**: 创建车票订单，用于用户选择车票和乘客信息后生成订单

### 请求参数

```json
{
  "ticketId": 1,
  "seatType": 2,
  "amount": 1106.4,
  "passengers": [
    {
      "name": "张三",
      "passengerType": 1,
      "seatPosition": "A"
    },
    {
      "name": "李四",
      "passengerType": 2,
      "seatPosition": "B"
    }
  ]
}
```

### 请求参数说明

- `ticketId`: 车票 ID (必填) - 从车票搜索结果中获取的车票唯一标识符
- `seatType`: 座位类型 (必填) - 座位类型代码，0-商务座，1-一等座，2-二等座
- `amount`: 订单总金额 (必填) - 根据乘客类型计算的订单总价，成人原价，其他类型8折
- `passengers`: 乘客信息列表 (必填) - 包含所有乘客的详细信息
  - `name`: 乘客姓名 (必填) - 乘客的真实姓名
  - `passengerType`: 乘客类型 (必填) - 乘客类型代码，1-成人，2-儿童，3-学生，4-老人
  - `seatPosition`: 座位位置 (可选) - 座位位置偏好，A-靠窗，B-中间，C-过道，D-过道，F-靠窗

### 响应格式

```json
{
  "success": true,
  "code": 200,
  "message": "订单创建成功",
  "data": {
    "id": 1,
    "orderId": 202401150001,
    "totalAmount": 1106.0,
    "status": 1,
    "expireTime": "2024-01-15T15:30:00",
    "createTime": "2024-01-15T15:00:00",
    "orderDetail": {
      "trainId": 1,
      "trainName": "复兴号",
      "departureStation": "北京南",
      "arrivalStation": "上海虹桥",
      "departureTime": "2024-01-15T08:00:00",
      "arrivalTime": "2024-01-15T12:30:00",
      "seatType": 2,
      "passengerCount": 2,
      "unitPrice": 553.0
    },
    "passengers": [
      {
        "name": "张三",
        "passengerType": 1,
        "seatPosition": "A"
      },
      {
        "name": "李四",
        "passengerType": 1,
        "seatPosition": "B"
      }
    ]
  }
}
```

### 响应参数说明

- `success`: 请求是否成功 - true 表示成功，false 表示失败
- `code`: 响应状态码 - 200 表示成功，其他值表示错误
- `message`: 响应消息 - 描述请求处理结果
- `data`: 响应数据对象
  - `id`: 主键 ID - 数据库中的主键标识符
  - `orderId`: 订单 ID - 系统生成的订单唯一标识符
  - `totalAmount`: 订单总金额 - 包含所有乘客的票价总和
  - `status`: 订单状态 - 订单的当前状态代码（1-待支付，2-已支付，3-已取消等）
  - `expireTime`: 订单过期时间 - 订单支付的截止时间，格式：ISO 8601
  - `createTime`: 订单创建时间 - 订单生成的时间，格式：ISO 8601
  - `orderDetail`: 订单详情 - 包含车票相关信息
    - `trainId`: 列车 ID - 列车的唯一标识符
    - `trainName`: 列车名称 - 列车的名称
    - `departureStation`: 出发站 - 出发车站名称
    - `arrivalStation`: 到达站 - 到达车站名称
    - `departureTime`: 出发时间 - 列车出发时间，格式：ISO 8601
    - `arrivalTime`: 到达时间 - 列车到达时间，格式：ISO 8601
    - `seatType`: 座位类型 - 座位类型代码，0-商务座，1-一等座，2-二等座
    - `passengerCount`: 乘客数量 - 本订单包含的乘客总数
    - `unitPrice`: 单价 - 每张票的价格
  - `passengers`: 乘客列表 - 包含所有乘客信息
    - `name`: 乘客姓名 - 乘客的真实姓名
    - `passengerType`: 乘客类型 - 乘客类型代码，1-成人，2-儿童，3-学生，4-老人
    - `seatPosition`: 座位位置 - 座位位置偏好

### 错误响应格式

```json
{
  "success": false,
  "code": 400,
  "message": "座位不足，请重新选择",
  "data": null
}
```

### 常见错误码

| 错误码 | 错误信息         | 说明                         |
| ------ | ---------------- | ---------------------------- |
| 200    | 订单创建成功     | 请求成功处理                 |
| 400    | 参数错误         | 请求参数不正确或缺失必要参数 |
| 401    | 未登录           | 用户未登录或登录已过期       |
| 403    | 权限不足         | 用户没有创建订单的权限       |
| 404    | 车票不存在       | 指定的车票 ID 不存在         |
| 409    | 座位不足         | 所选座位类型余票不足         |
| 422    | 乘客信息验证失败 | 乘客信息不符合要求           |
| 500    | 服务器内部错误   | 系统异常，请稍后重试         |

### 业务规则

1. **乘客数量限制**: 单个订单最多支持 6 名乘客
2. **座位预分配**: 系统会根据用户选择的座位类型和位置偏好进行座位预分配
3. **座位分配**: 系统会自动为乘客分配座位，优先考虑用户的座位位置偏好
4. **订单有效期**: 订单创建后30分钟内必须完成支付，否则自动取消
5. **身份验证**: 乘客姓名必须真实有效
6. **重复订单**: 同一用户在同一车次同一时间不能创建重复订单
7. **价格计算**: 订单金额根据乘客类型自动计算，成人按原价收费，儿童、学生、老人享受8折优惠

### 前端调用示例

```javascript
import { createOrder } from "@/api/order.js";

// 创建订单
const handleCreateOrder = async () => {
  const orderData = {
    ticketId: 1,
    seatType: 2,
    amount: 1106.4, // 成人553 + 儿童553*0.8 = 553 + 442.4 = 995.4
    passengers: [
      {
        name: "张三",
        passengerType: 1,
        seatPosition: "A"
      },
      {
        name: "李四",
        passengerType: 2,
        seatPosition: "B"
      }
    ]
  },
    ],
  };

  try {
    const response = await createOrder(orderData);

    if (response.success) {
      console.log("订单创建成功:", response.data.orderId);
      // 跳转到支付页面
      router.push({
        path: "/ticket-payment",
        query: {
          orderId: response.data.orderId,
        },
      });
    } else {
      console.error("订单创建失败:", response.message);
    }
  } catch (error) {
    console.error("网络错误:", error);
  }
};
```

### 注意事项

1. **数据验证**: 前端需要验证乘客信息的完整性和有效性
2. **错误处理**: 需要对各种错误情况进行适当的用户提示
3. **支付跳转**: 订单创建成功后应立即引导用户进行支付
4. **订单状态**: 需要定期检查订单状态，处理超时订单
5. **状态码处理**: 注意订单状态和乘客类型使用数字代码，需要在前端进行相应的转换显示
6. **价格计算**: 前端需要根据乘客类型正确计算订单总金额，确保与后端计算逻辑一致
