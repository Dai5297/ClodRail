# 积分商城数据库设计说明

## 商品表设计

### 表名：item

该表用于存储积分商城的商品信息。

### 表结构

```sql
CREATE TABLE IF NOT EXISTS `item` (
  `id` bigint NOT NULL AUTO_INCREMENT COMMENT '商品id',
  `name` varchar(200) CHARACTER SET utf8mb4 COLLATE utf8mb4_general_ci NOT NULL COMMENT '商品名称',
  `price` int NOT NULL DEFAULT 0 COMMENT '积分价格',
  `stock` int UNSIGNED NOT NULL COMMENT '库存数量',
  `image` varchar(500) CHARACTER SET utf8mb4 COLLATE utf8mb4_general_ci DEFAULT NULL COMMENT '商品主图',
  `images` text CHARACTER SET utf8mb4 COLLATE utf8mb4_general_ci DEFAULT NULL COMMENT '商品多图（JSON数组字符串）',
  `category` varchar(50) CHARACTER SET utf8mb4 COLLATE utf8mb4_general_ci DEFAULT NULL COMMENT '分类编码',
  `brand` varchar(100) CHARACTER SET utf8mb4 COLLATE utf8mb4_general_ci DEFAULT NULL COMMENT '品牌名称',
  `spec` text CHARACTER SET utf8mb4 COLLATE utf8mb4_general_ci DEFAULT NULL COMMENT '规格信息',
  `description` text CHARACTER SET utf8mb4 COLLATE utf8mb4_general_ci DEFAULT NULL COMMENT '商品描述',
  `sold` int UNSIGNED DEFAULT 0 COMMENT '销量',
  `comment_count` int UNSIGNED DEFAULT 0 COMMENT '评论数',
  `isAD` tinyint(1) DEFAULT 0 COMMENT '是否推广 0-否 1-是',
  `tags` varchar(200) CHARACTER SET utf8mb4 COLLATE utf8mb4_general_ci DEFAULT NULL COMMENT '标签（逗号分隔）',
  `status` tinyint DEFAULT 1 COMMENT '状态 1-正常 2-下架 3-删除',
  `sort_order` int DEFAULT 0 COMMENT '排序权重',
  `create_time` datetime DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
  `update_time` datetime DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
  `creater` bigint DEFAULT NULL COMMENT '创建人',
  `updater` bigint DEFAULT NULL COMMENT '修改人',
  PRIMARY KEY (`id`),
  KEY `idx_status_category` (`status`, `category`),
  KEY `idx_price` (`price`),
  KEY `idx_update_time` (`update_time`),
  KEY `idx_sort_order` (`sort_order`)
) ENGINE=InnoDB AUTO_INCREMENT=100002672305 DEFAULT CHARSET=utf8mb4 COMMENT='积分商城商品表';
```

### 字段说明

| 字段名 | 数据类型 | 是否必填 | 默认值 | 说明 |
|--------|---------|---------|--------|------|
| id | bigint | 是 | 自增 | 商品唯一标识 |

| brand | varchar(100) | 否 | NULL | 品牌名称 |

## 查询SQL示例

### 1. 获取商品列表（分页+筛选）

```sql
SELECT 
  id, name, price, image, images, stock, category, brand, 
  spec, description, sold, comment_count, isAD, tags, 
  status, sort_order, create_time, update_time
FROM item
WHERE status = 1 
  AND (category = ? OR ? IS NULL)
  AND name LIKE CONCAT('%', ?, '%')
ORDER BY 
  CASE WHEN ? = 'points-asc' THEN price END ASC,
  CASE WHEN ? = 'points-desc' THEN price END DESC,
  CASE WHEN ? = 'popularity' THEN sold END DESC,
  sort_order DESC,
  create_time DESC
LIMIT ?, ?;
```

### 2. 获取商品详情

```sql
SELECT 
  id, name, price, image, images, stock, category, brand, 
  spec, description, sold, comment_count, isAD, tags, 
  status, sort_order, create_time, update_time
FROM item
WHERE id = ? AND status = 1;
```

### 3. 扣减库存（兑换商品时）

```sql
UPDATE item 
SET stock = stock - ?,
    sold = sold + ?,
    update_time = NOW()
WHERE id = ? 
  AND stock >= ?
  AND status = 1;
```

## 注意事项

1. **价格字段**：`price` 字段直接存储积分数值，不需要 `original_price` 字段
2. **字符集**：使用 utf8mb4 支持 emoji 等特殊字符
3. **库存并发**：扣减库存时需要使用乐观锁或悲观锁防止超卖
4. **软删除**：删除商品时设置 status=3，不物理删除数据
5. **图片存储**：images 字段存储 JSON 数组字符串，前端需要解析
6. **标签处理**：tags 字段存储逗号分隔字符串，前端需要 split 处理
7. **索引优化**：status 和 category 的组合索引可以加速常用查询

## 扩展建议

如果后续需要支持更复杂的功能，可以考虑以下扩展：

1. **商品分类表**：将 category 独立为表，支持多级分类
2. **商品SKU表**：支持同一商品的多个规格（如颜色、尺寸）
3. **兑换记录表**：记录用户兑换历史
4. **商品评价表**：用户对兑换商品的评价
5. **库存流水表**：记录库存变动历史
