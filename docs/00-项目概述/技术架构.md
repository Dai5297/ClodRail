## 架构分层

```
┌─────────────────────────────────────────┐
│              前端层                      │
│   用户端 Web    │    管理端 Web          │
└─────────────────────────────────────────┘
                  ↓
┌─────────────────────────────────────────┐
│           接入层（网关）                  │
│   路由转发 │ JWT鉴权 │ 限流 │ 跨域       │
└─────────────────────────────────────────┘
                  ↓
┌─────────────────────────────────────────┐
│          业务微服务层                    │
│  user │ ticket │ order │ mall │ assistant│
└─────────────────────────────────────────┘
                  ↓
┌─────────────────────────────────────────┐
│           基础设施层                     │
│ Nacos│MySQL│Redis│RabbitMQ│Seata│ES    │
└─────────────────────────────────────────┘
```

## 数据模型

### 核心实体关系

```
USER ──┬── ORDER ──── ORDER_PASSENGER
       │      │
       │      └── TRAIN/TICKET
       │
       ├── POINTS_RECORD
       │
       └── POINTS_ORDER ── POINTS_PRODUCT
```

### 主要数据表

- **用户相关**：user, admin, role, permission
- **车票相关**：station, line, train, ticket, seat
- **订单相关**：order, order_passenger
- **商城相关**：points_product, points_order, points_record
