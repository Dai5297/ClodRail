# ClodRail 项目介绍

## 项目总览

ClodRail 是一套仿 12306 的铁路票务系统，采用前后端分离与微服务架构，面向高并发购票与实时客服场景。

## 整体架构

```
┌─────────────────────────────────────────────────────────────────┐
│                         前端层                                   │
│     用户端 (Vue3 + Element Plus)    管理端 (Vue3 + Element Plus) │
└─────────────────────────────────────────────────────────────────┘
                                ↓
┌─────────────────────────────────────────────────────────────────┐
│                    统一网关 (Spring Cloud Gateway)               │
│              路由转发 │ JWT 鉴权 │ 限流 │ 跨域处理                │
└─────────────────────────────────────────────────────────────────┘
                                ↓
┌──────────┬──────────┬──────────┬──────────┬──────────┐
│ 用户服务  │ 车票服务  │ 订单服务  │ 商城服务  │ 客服服务  │
│ :18081   │ :18082   │ :18083   │ :18086   │ :18084   │
└──────────┴──────────┴──────────┴──────────┴──────────┘
                                ↓
┌─────────────────────────────────────────────────────────────────┐
│                        基础设施层                                │
│   Nacos │ MySQL │ Redis │ RabbitMQ │ Seata │ Elasticsearch      │
└─────────────────────────────────────────────────────────────────┘
```

## 前端应用

- `rs-user-web`：用户端（搜索购票、订单支付、积分商城、AI 客服）
- `rs-admin-web`：管理端（运营仪表盘、车次/线路/车票、订单、权限、客服回复）

## 后端服务

| 服务 | 模块 | 职责 |
|------|------|------|
| rs-gateway | 网关 | 统一入口，路由转发、鉴权、限流 |
| rs-user | 用户服务 | 用户管理与权限控制 |
| rs-ticket | 车票服务 | 车站、线路、座位与车票库存 |
| rs-order | 订单服务 | 订单创建、支付、退改签 |
| rs-mall | 商城服务 | 积分商城，商品展示与兑换 |
| rs-assistant | 客服服务 | AI 客服助手，WebSocket 实时通信 |

## 核心中间件

- **MySQL**：业务数据持久化
- **Redis**：缓存、原子扣减与并发控制
- **RabbitMQ**：消息驱动的异步处理
- **Elasticsearch**：商品搜索与检索
- **Seata**：TCC 分布式事务
- **Netty**：WebSocket 长连接

## 亮点功能

### 高并发下单

1. 用户搜索车次，网关路由至 `rs-ticket` 查询余票
2. 用户选座下单，网关路由至 `rs-order`
3. 订单服务根据车票类型走不同流程：
   - 普通车票：直接检查余票 → 分配座位 → 写入订单
   - 热门车票：Redis + Lua 原子预扣 → TCC 事务落库
4. 订单成功后通过 RabbitMQ 异步处理积分发放

### 热门秒杀一致性保障

- **Redis + Lua**：余票扣减原子性
- **Seata TCC**：两阶段事务（Try/Commit/Rollback）
- **RabbitMQ**：异步落库与积分发放

### 实时通信与客服

- Netty + WebSocket 长连接
- AI 智能问答 + 人工客服转接
- 会话记忆持久化

### RBAC 权限管理

- Spring Security 实现角色权限控制
- 管理端可配置用户、角色与权限

## 技术栈

- **后端**：Spring Boot 3.5.5, Spring Cloud 2025.0.0, Spring Cloud Alibaba
- **前端**：Vue 3, Vite, Element Plus, TailwindCSS
- **中间件**：MySQL, Redis, RabbitMQ, Elasticsearch, Seata, Netty
