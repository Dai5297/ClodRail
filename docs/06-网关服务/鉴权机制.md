# 网关鉴权机制

## 认证流程

```
1. 客户端请求携带 Authorization Header
2. 网关 SecurityFilter 拦截请求
3. 检查是否在白名单（放行）
4. 解析 Bearer Token
5. 从 Redis 获取 JWT
6. 验证 JWT 并解析用户信息
7. 将用户信息写入请求属性
8. 转发到下游服务
```

## 请求头

```
Authorization: Bearer {token}
```

## 白名单配置

以下路径无需认证：

- `/customer/auth/login`
- `/customer/auth/register`
- `/admin/auth/login`
- `/doc.html`
- `/swagger-resources/**`
- `/v3/api-docs/**`

## Redis Key 设计

| Key 格式 | 说明 |
|----------|------|
| `user:login:token:{uuid}` | 用户 Token |
| `admin:login:token:{uuid}` | 管理员 Token |

## JWT 结构

```json
{
  "sub": "{userId}",
  "exp": 1703123456,
  "iat": 1703116256
}
```

## 权限校验

管理端请求 (`/admin/**`) 额外进行权限校验：

1. 从 Redis 获取角色权限列表
2. 匹配 `METHOD-pathPattern`
3. 超级管理员直接放行

## 错误响应

### 401 未授权

```json
{
  "code": 401,
  "message": "未登录或登录已过期"
}
```

### 403 权限不足

```json
{
  "code": 403,
  "message": "无权限访问该资源"
}
```
