# 快速启动指南

这份文档面向第一次在本地运行 ClodRail 的读者。目标不是覆盖所有部署方式，而是尽快把项目跑起来。

## 1. 环境准备

先准备这些基础组件：

- MySQL 8.0+
- Redis 7.0+
- RabbitMQ 3.12+
- Nacos 2.x

可选组件：

- Seata 1.6+，用于分布式事务场景
- Elasticsearch 8.x，供积分商城搜索使用

详细版本建议见 [环境要求](环境要求.md)。

## 2. 初始化数据库

### 2.1 创建 Nacos 数据库并导入共享配置

```bash
mysql -u root -p -e "CREATE DATABASE nacos DEFAULT CHARACTER SET utf8mb4;"
mysql -u root -p nacos < docs/99-部署运维/nacos.sql
```

`docs/99-部署运维/nacos.sql` 会导入共享配置，包括：

- MySQL 数据源配置
- Redis 配置
- RabbitMQ 配置
- Seata 配置
- Knife4j 配置
- 支付配置
- LangChain4j 配置

### 2.2 创建业务数据库

```bash
mysql -u root -p -e "
CREATE DATABASE rs_user DEFAULT CHARACTER SET utf8mb4;
CREATE DATABASE rs_ticket DEFAULT CHARACTER SET utf8mb4;
CREATE DATABASE rs_order DEFAULT CHARACTER SET utf8mb4;
CREATE DATABASE rs_mall DEFAULT CHARACTER SET utf8mb4;
CREATE DATABASE seata DEFAULT CHARACTER SET utf8mb4;
"
```

## 3. 启动基础设施

```bash
# 启动 Nacos（单机模式）
cd nacos/bin
./startup.sh -m standalone
```

MySQL、Redis、RabbitMQ 可以使用本地安装或 Docker 方式启动。

## 4. 检查并补充本地配置

项目依赖 Nacos 中的共享配置，以及各服务本地环境变量或私有配置。第一次启动前，建议重点检查：

- MySQL 账号、密码、库名
- Redis 地址和密码
- RabbitMQ 地址和账号
- Nacos namespace
- 支付宝沙箱配置
- AI 模型 API Key

如果你只想先验证主流程，可以暂时不启用 Elasticsearch、Seata、AI 等扩展能力，但对应功能可能不可用。

## 5. 编译后端

```bash
cd RailwaySystem-Backend
mvn clean install -DskipTests
```

## 6. 启动后端服务

推荐顺序：

```bash
# 1. 网关
cd RailwaySystem-Backend/rs-gateway
mvn spring-boot:run

# 2. 用户服务
cd RailwaySystem-Backend/rs-service/rs-user
mvn spring-boot:run

# 3. 车票服务
cd RailwaySystem-Backend/rs-service/rs-ticket
mvn spring-boot:run

# 4. 订单服务
cd RailwaySystem-Backend/rs-service/rs-order
mvn spring-boot:run

# 5. 商城服务
cd RailwaySystem-Backend/rs-service/rs-mall
mvn spring-boot:run

# 6. 客服服务
cd RailwaySystem-Backend/rs-service/rs-assistant
mvn spring-boot:run
```

## 7. 启动前端

```bash
# 用户端
cd RailwaySystem-Frontend/rs-user-web
npm install
npm run dev

# 管理端
cd RailwaySystem-Frontend/rs-admin-web
npm install
npm run dev
```

## 8. 访问验证

| 服务 | 地址 |
|------|------|
| Nacos 控制台 | http://localhost:8848/nacos |
| API 文档 | http://localhost:18080/doc.html |
| 用户端 | http://localhost:5173 |
| 管理端 | http://localhost:5174 |

Nacos 默认账号一般为 `nacos / nacos`。

## 9. 常见启动判断

你可以通过下面几点快速判断项目是否基本启动成功：

- Nacos 中能看到共享配置
- 网关 `18080` 可以访问
- 用户端或管理端可以正常打开首页
- 网关 API 文档页面能加载

如果其中某一步失败，优先检查基础设施是否已启动、Nacos namespace 是否正确、数据库账号密码是否和共享配置一致。
