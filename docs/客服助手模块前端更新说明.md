# 客服助手模块前端更新说明

## 更新日期

2025-11-05

## 更新概述

根据《客服助手 API 接口文档》规范，更新前端代码中的所有客服助手相关接口路径，统一添加 `/user` 路径前缀。

---

## 修改文件清单

### 1. API 接口文件

**文件路径**: `rs-user-web/src/api/assistant.js`

#### 修改内容

| 函数名                     | 原路径                           | 新路径                                | 说明                    |
| -------------------------- | -------------------------------- | ------------------------------------- | ----------------------- |
| `sendAIMessage`            | `/assistant/chat`                | `/api/user/assistant/chat`            | AI 智能问答（流式响应） |
| `getChatSessions`          | `/assistant/sessions`            | `/user/assistant/sessions`            | 获取会话历史列表        |
| `getChatHistory`           | `/assistant/history/{sessionId}` | `/user/assistant/history/{sessionId}` | 获取会话消息历史        |
| `createChatSession`        | `/assistant/session`             | `/user/assistant/session`             | 创建新会话              |
| `deleteChatSession`        | `/assistant/session/{sessionId}` | `/user/assistant/session/{sessionId}` | 删除会话                |
| `getCustomerServiceStatus` | `/assistant/service/status`      | `/user/assistant/service/status`      | 获取客服状态            |
| `requestTransfer`          | `/assistant/service/transfer`    | `/user/assistant/service/transfer`    | 申请转接人工客服        |

#### 特殊说明

1. **`sendAIMessage` 函数**

   - 使用原生 `fetch` API，需要添加完整路径前缀 `/api/user/assistant/chat`
   - `/api` 前缀用于触发 Vite 代理配置
   - 其他函数使用 axios，会自动添加 baseURL (`/api`)

2. **代理配置**
   - Vite 配置 (`vite.config.js`) 将 `/api` 代理到 `http://localhost:18080`
   - 代理会自动去除 `/api` 前缀
   - 最终请求到后端的路径格式：`http://localhost:18080/user/assistant/*`

## 接口路径规范

### REST API 接口

所有客服助手相关接口统一使用 `/user/assistant/` 路径前缀：

```
/user/assistant/chat                    - AI智能问答
/user/assistant/sessions                - 获取会话列表
/user/assistant/history/{sessionId}     - 获取会话历史
/user/assistant/session                 - 创建/删除会话
/user/assistant/service/status          - 获取客服状态
/user/assistant/service/transfer        - 转接人工客服
```

### WebSocket 接口

WebSocket 连接地址保持不变：

```
ws://localhost:18080/ws/assistant?sessionId={sessionId}&token={token}
```

## 前端调用示例

### 使用 axios (通过 request 工具)

```javascript
import { getChatSessions } from "@/api/assistant";

// 自动添加 /api 前缀，通过代理转发到后端
const response = await getChatSessions();
```

**实际请求路径**：`http://localhost:18080/user/assistant/sessions`

### 使用 fetch (流式响应)

```javascript
import { sendAIMessage } from "@/api/assistant";

// 需要手动添加 /api 前缀
await sendAIMessage("如何购票？", sessionId, (chunk) => {
  console.log("AI回复:", chunk);
});
```

**实际请求路径**：`http://localhost:18080/user/assistant/chat`

## 配置说明

### 1. Vite 代理配置 (vite.config.js)

```javascript
server: {
  proxy: {
    '/api': {
      target: 'http://localhost:18080',
      changeOrigin: true,
      rewrite: (path) => path.replace(/^\/api/, '')
    }
  }
}
```

**工作原理**：

- 前端请求：`/api/user/assistant/chat`
- 代理转发：`http://localhost:18080/user/assistant/chat`

### 2. Axios 配置 (src/utils/request.js)

```javascript
const request = axios.create({
  baseURL: "/api",
  timeout: 5000,
});
```

**自动处理**：

- 所有使用 `request` 的 API 调用会自动添加 `/api` 前缀
- 自动添加 Authorization Token
- 统一处理错误响应

## 注意事项

### 1. 开发环境

- 后端服务必须运行在 `http://localhost:18080`
- 前端开发服务器会自动代理 API 请求
- WebSocket 连接直接连接到后端，不经过代理

### 2. 生产环境

- 需要配置 Nginx 反向代理
- 确保 `/user/assistant/*` 路径正确转发到后端服务
- WebSocket 连接需要配置 Upgrade 和 Connection 头

### 3. 兼容性说明

- 流式响应需要浏览器支持 `ReadableStream` API
- WebSocket 需要浏览器支持 WebSocket API
- 建议使用现代浏览器（Chrome 90+, Firefox 88+, Safari 14+）
